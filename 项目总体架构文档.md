> 项目代号：**宝青坊（BaoQingFang）**
> 文档版本：v1.0
> 用途：作为开发蓝图，指导后续代码实现、素材准备和运营落地

## 第一章 项目定位

### 1.1 项目本质

一个**事件驱动的智能素材调度系统**：

- **输入**：直播间的实时事件（用户进入、弹幕、点赞、送礼、下单）
- **处理**：意图识别 → 风格匹配 → 内容生成 → 多通道输出
- **输出**：角色视频片段 + TTS语音 + 屏幕弹窗 + 商品展示
- **载体**：浏览器页面（被OBS采集后推流到抖音/视频号）

### 1.2 核心价值

| 维度 | 传统真人直播 | 传统数字人直播 | 本项目 |
|---|---|---|---|
| 24小时运营 | ❌ | ✅ | ✅ |
| 互动性 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐ |
| 场景沉浸感 | ⭐⭐ | ⭐ | ⭐⭐⭐⭐ |
| 风格多样性 | ⭐⭐⭐ | ⭐ | ⭐⭐⭐⭐⭐ |
| 运营成本 | 高 | 低 | 低 |
| 同质化风险 | 中 | 高 | 低 |

### 1.3 商业模型

- **品类聚焦**：茶叶、香薰、文房文创（古风人设匹配品类）
- **客单价**：¥50-300（冲动消费区间）
- **流量来源**：搜索流量 + 短视频引流 + 粉丝沉淀
- **收入构成**：CPS佣金 + 自营商品差价

## 第三章 事件处理体系

### 3.1 事件类型与优先级

| 优先级 | 事件类型 | 描述 | 响应策略 | 中断当前？ |
|---|---|---|---|---|
| **1** | `order_placed` | 用户下单成功 | 立即播放感谢+特效 | ✅ 中断 |
| **2** | `gift_big` | 大额礼物（嘉年华等）| 全屏特效+长感谢 | ✅ 中断 |
| **3** | `urgent_question` | 询价/库存等紧急 | 优先响应 | ⚠️ 等当前结束 |
| **4** | `product_query` | 商品咨询 | 标准流程 | ❌ 排队 |
| **5** | `style_switch` | 切换对话风格 | 立即切换 | ⚠️ 等当前结束 |
| **6** | `gift_small` | 小礼物（爱心点赞）| 累计后批量感谢 | ❌ 合并 |
| **7** | `general_chat` | 普通闲聊 | 排队，可丢弃 | ❌ 排队 |
| **8** | `user_enter` | 新用户进入 | 30秒批量欢迎 | ❌ 合并 |
| **9** | `like_received` | 点赞 | 计数定时反馈 | ❌ 合并 |

### 3.2 流量控制规则

```
全局规则：
- 队列总长度上限：50条
- 超出时丢弃低优先级
- 每分钟AI调用上限：30次（控制成本）

分级处理上限：
- 高优先级（1-3）：无限制
- 中优先级（4-5）：20条/分钟
- 低优先级（6-9）：5条/分钟

合并去重：
- 同SKU重复询问3条内合并："好几位客官都在问这茶..."
- 用户10分钟内不重复欢迎
- 相同问候语避免连续使用
```

## 第四章 核心销售场景设计

### 4.1 场景清单

| 编号 | 场景名 | 触发 | 优先级 |
|---|---|---|---|
| S01 | 用户进入欢迎 | `user_enter` | 低 |
| S02 | 商品询问 | 弹幕含SKU关键词 | 中 |
| S03 | 价格询问 | 弹幕含价格关键词 | 中 |
| S04 | 下单感谢 | `order_placed` | 最高 |
| S05 | 礼物感谢 | `gift_*` | 高 |
| S06 | 负面弹幕（狐狸头）| 检测脏话/恶意 | 中 |
| S07 | 风格切换 | 用户指定风格 | 中 |
| S08 | 促单/逼单 | 定时/在线人数阈值 | 中 |
| S09 | 闲聊互动 | 无意图弹幕 | 低 |
| S10 | 对比咨询 | "A和B哪个好" | 中 |
| S11 | 政策咨询 | 物流/售后 | 中 |
| S12 | 库存提醒 | "还有货吗" | 中 |

### 4.2 各场景动作序列（标准模板）

#### S01：用户进入欢迎

```
触发：累计30秒内进入用户
执行：
  [0.0s] 角色播放 greet 动画
  [0.5s] TTS：根据用户数选择话术
         - 1-2人：点名欢迎
         - 3-5人：批量欢迎
         - 6+人：群体欢迎
  [3.0s] 回到 idle
```

#### S02：商品询问（完整流程）

```
触发：弹幕匹配SKU关键词
执行：
  [0.0s] 角色"思考"表情（0.5秒）
  [0.5s] TTS："客官好眼光，这便为您取来"
  [1.0s] 播放 fetch_turn 动画（转身）
  [3.0s] 货架对应SKU闪烁高亮
  [3.5s] 播放 fetch_return 动画（拿回）
  [5.0s] 手部显示商品图，桌面显示大图
  [5.5s] 桌面屏幕播放介绍视频
  [6.0s] TTS 介绍 + 真实价格
  [6.0s] 同步弹出价格信息卡（类型B）
  [12s] 引导购买："小黄车第X个"
  [12s] 弹出操作引导卡（类型C，含箭头）
  [15s] 回到 idle
```

#### S03：价格询问（防AI幻觉）

```
触发：用户问价格
处理：
  1. 解析数量（1盒/3盒）
  2. 匹配最优价格档
  3. 从配置读取真实价格（绝不让AI编造）
  4. 模板填充 → AI润色 → 后置校验
执行：
  - 口头：角色说出价格
  - 弹窗：信息卡片显示价格对比
  - 视觉：高亮当前推荐档位
```

#### S04：下单感谢（最高优先级）

```
触发：抖音回调订单事件
动作：立即中断当前
执行：
  [0.0s] 角色播放 surprise 动画
  [0.0s] 全屏金币雨特效
  [0.5s] 播放下单提示音
  [1.0s] TTS点名感谢
  [1.0s] 弹窗显示"已售X单"计数（紧迫感）
  [5.0s] 回到 idle，继续之前流程
```

#### S05-S12：略（详见后续完整设计）

## 第五章 角色与风格系统

### 5.1 角色基础设定

- **名称**：宝青坊主
- **背景**：千年狐妖，开神秘杂货铺
- **核心特征**：表面温柔，腹黑狡黠，生气变狐狸头

### 5.2 风格矩阵（5种）

| 风格ID | 名称 | 语气 | 视觉 | 触发词 |
|---|---|---|---|---|
| `classical` | 古风坊主（默认）| 古韵客气，"客官""呢""呀" | 人形温和 | "正常""古风" |
| `cute` | 嗲嗲甜妹 | "啦""嘛""呢"尾音、撒娇 | 歪头眨眼 | "可爱""嗲""甜" |
| `dominant` | 高冷御姐 | 简短命令，"本坊主""听着" | 抱臂挑眉 | "御姐""霸气" |
| `tsundere` | 傲娇毒舌 | 嘴硬心软，"哼""笨蛋" | 扭头扇遮 | "傲娇""嘴硬" |
| `mysterious` | 神秘蛊惑 | 气音低语，暗示性 | 狐狸头常驻 | "神秘""狐狸精" |

### 5.3 风格切换机制

```
用户："你好凶啊"
→ 检测负面 → 古风回复 + 主动提议切换
→ "哎呀客官误会啦~人家天生的呢，要不换个嗲嗲的？"

用户："来个高冷的"
→ 匹配 dominant → 立即切换
→ 播放过渡台词："哼，要求真多。"
→ 后续对话使用 dominant prompt 和 voice
```

### 5.4 每种风格所需视频素材

每种风格需要的核心动作（最少8个）：

```
{style}_idle              # 待机循环
{style}_greet             # 打招呼
{style}_fetch_turn        # 转身拿物品
{style}_fetch_return      # 拿回展示
{style}_present           # 介绍商品
{style}_thanks_order      # 感谢下单
{style}_thanks_gift       # 感谢礼物
{style}_farewell          # 送别
```

**进阶素材（每种风格再加6个）**：
```
{style}_surprise          # 惊讶（下单时）
{style}_thinking          # 思考
{style}_laugh             # 笑
{style}_pout              # 噘嘴/不满
{style}_angry             # 生气（古风专属变狐狸）
{style}_special           # 风格专属特色动作
```

**总计**：3种核心风格 × 14动作 = **42个视频片段**（MVP阶段做24个）

## 第六章 多通道输出体系

### 6.1 四个输出通道

| 通道 | 内容 | 设备 |
|---|---|---|
| **视频通道** | 角色动画 | 显示器（OBS采集）|
| **音频通道** | TTS语音 + BGM + 音效 | 系统音频（OBS混音）|
| **弹窗通道** | 信息卡片 / 操作引导 | 显示器（叠加在画面上）|
| **UI通道** | 商品图 / 价签 / 特效 | 显示器（场景层级）|

### 6.2 弹窗系统（4种类型）

#### 类型A：信息气泡（轻量）
- 位置：角色头部右上方
- 显示：纯文字，3-5秒自动消失
- 用途：简短信息（价格、保质期）

#### 类型B：信息卡片（中等）
- 位置：屏幕右侧固定区域
- 显示：标题+内容+图标，停留10秒
- 用途：详细参数、价格对比

#### 类型C：操作引导卡（强引导）
- 位置：屏幕中下方
- 显示：商品图+价格+"点击小黄车"箭头
- 用途：转化关键节点

#### 类型D：模拟输入框（互动感）
- 位置：屏幕底部
- 显示：聊天框样式，逐字打出
- 用途：复杂咨询（对比、推荐）

### 6.3 弹窗触发规则

| 用户咨询类型 | 弹窗类型 |
|---|---|
| 价格/规格 | A（气泡）|
| 参数/配料/功效 | B（卡片）|
| 怎么买/链接 | C（引导卡）|
| 对比/推荐 | D（聊天框）|
| 物流/售后 | B（卡片）|
| 普通闲聊 | 无 |

### 6.4 多通道协同时序示例

**场景**：用户问"3盒多少钱"

```
时间轴：
[0.0s] 视频：fetch_turn 启动
[0.0s] 弹窗：B类信息卡淡入（含价格档对比）
[0.5s] 音频：TTS"客官好眼力，三盒囤货装..."
[3.0s] 视频：fetch_return
[3.0s] UI：手部商品图显示
[6.0s] 弹窗：B类淡出
[6.0s] 弹窗：C类操作引导卡显示（箭头指小黄车）
[6.0s] 音频：TTS"小黄车第3个，喜欢的客官快下单"
[10s]  视频：回到 idle
[14s]  弹窗：C类淡出
```

### 11.2 业务模式确认

> **本项目最终确定模式：模式B（一件代发 / 多平台分销）**

#### 三种模式对比（参考）

### 11.7 订单回调对接

1. 登录聚水潭 → 设置 → 开发者中心

3. 应用授权 → 选择需要的接口权限：
   - 商品查询
   - 库存查询
   - 订单查询/订阅
4. 店铺绑定 → 确保抖音/淘宝店铺已接入聚水潭

# 2. 库存查询

  - sku_ids: ["JST001", "JST002"]

# 3. 订单查询（拉取模式）

  - shop_id: "your_shop_id"

# 4. 订单事件订阅（推送模式，推荐）

  - event_types: ["order.created"]

#### 数据同步策略

```
启动时（首次）：
1. 全量拉取商品列表
2. 全量拉取库存
3. 写入本地缓存

运行时（增量）：
1. 商品同步：每小时一次（变化频率低）
2. 库存同步：每5分钟一次（避免超卖）
3. 订单同步：
   - 优先：Webhook推送（实时）
   - 兜底：每30秒轮询新订单（防漏）

- API限流：实现请求队列+退避策略
- Token过期：自动刷新token
- 网络异常：本地缓存+断点续传
```
